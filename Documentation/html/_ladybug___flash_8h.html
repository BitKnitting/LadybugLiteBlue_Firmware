void ladybug_flash_write ( flash_rw_t  what_data_to_write,
uint8_t *  p_bytes_to_write,
pstorage_size_t  num_bytes_to_write,
void(*)(uint32_t err_code)  did_flash_action 
)

When the Ladybug needs to store info, it calls the flash_write routine.

This routine assumes the flash storage to be used has been initialized by a call to flash_init. Match the handle to flash storage to the info the caller wants to write.

Note
As directed by the nRF51822 documentation, the flash storage is first cleared before a write to flash happens.
Parameters
what_data_to_writeWhether to write out plant info, calibration values, or the device name.
p_bytes_to_writeA pointer to the bytes to be written to flash.
num_bytes_to_writeThe number of bytes to write to flash
did_flash_actionFunction caller passes in to be informed of the outcome of the Flash request. The pointer to the function must be valid.
